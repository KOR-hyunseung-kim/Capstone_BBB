[POWER & GND DISTRIBUTION]

배터리 3.7V (+)
    ↓
    ├─→ ESP32-S3 5V IN
    ├─→ [1kΩ] (분압기 상단)
    ├─→ MyoWare 2.0 +3.3V
    ├─→ MPU6050 VCC
    ├─→ MOSFET Drain (진동모터+)
    └─→ [선택] RGB LED VCC

배터리 3.7V (−) / GND
    ↑
    ├─← ESP32-S3 GND
    ├─← [2kΩ] (분압기 하단)
    ├─← MyoWare 2.0 GND
    ├─← MPU6050 GND
    ├─← MOSFET Source (진동모터−)
    └─← [선택] RGB LED GND


[SIGNAL WIRING DIAGRAM]

┌──────────────────────────────────────────────────────────────┐
│ ESP32-S3 (14-pin) ←→ 외부 부품 연결                         │
├──────────────────────────────────────────────────────────────┤

① GND   ──→ GND RAIL (배터리 −)
② 5V    ──→ POWER RAIL (배터리 +)

③ D0    ──→ MPU6050 SDA (I2C)
④ D1    ──→ MPU6050 SCL (I2C)

⑤ A0    ──→ 분압기 중점 ←─ MyoWare SIGNAL 입력
           (ADC 입력, 3.3V max)

⑥ D9    ──→ [100Ω 저항] ──→ MOSFET Gate
           (GPIO PWM, 진동모터 제어)

⑦ [예약] (또는 LED 신호)

└──────────────────────────────────────────────────────────────┘


[I2C BUS (D0, D1)]

        3.3V (POWER RAIL)
         ↓
        [R] (pull-up, 선택: 보통 내장)
         ↓
    ┌────┴────────┐
    │   I2C BUS   │
    │ SDA / SCL   │
    └─┬──────┬────┘
      │      │
      │      └─→ MPU6050 SCL (D1)
      └──────→ MPU6050 SDA (D0)

주소: 0x68 (또는 0x69, AD0 핀 상태에 따라)


[ADC INPUT (A0) - 분압기 회로]

    POWER RAIL (3.7V)
         ↓
       [1kΩ]
         ├───────→ ESP32-S3 A0 (ADC 입력)
       [2kΩ]      변환 범위: 0~3.3V
         ↓         매핑: 0V (신호없음) ~ 2.47V (신호있음)
    GND RAIL

MyoWare 신호 라우팅:
MyoWare 2.0 (EMG)
    ├─ +3.3V → POWER RAIL
    ├─ GND   → GND RAIL
    └─ SIG   → [옵션] 분압기 입력 또는 직접 A0
              (권장: 분압기로 잡음 억제)


[PWM OUTPUT (D9) - MOSFET 게이트]

  ESP32-S3 D9 (GPIO PWM)
         ↓
      [100Ω]    ← 스파이크 방지 저항
         ├─────→ MOSFET Gate
         │
      [1kΩ]    ← 안정화 저항 (Gate to Source)
         │
    GND RAIL

MOSFET 핀 정의 (AO3400/2N7000 기준):
    1. Gate (G)   ← 100Ω 저항으로부터
    2. Drain (D)  → POWER RAIL → 진동모터(+)
    3. Source (S) → GND RAIL ← 진동모터(−)


[진동모터 연결]

    진동모터
      (+) 극 ──→ MOSFET Drain
      (−) 극 ──→ MOSFET Source


[실제 배선 순서]

1️⃣ Power/GND Rail 연결 (굵은 선, 병렬 구조)
   배터리(+) ─────┬─────┬─────┬─────┬────
                  │     │     │     │
              ESP32  MYO  MPU  MOS  ...
                  │     │     │     │
   배터리(−) ─────┴─────┴─────┴─────┴────

2️⃣ I2C 신호선 (단선, 최단거리)
   ESP32 D0 ──→ MPU6050 SDA
   ESP32 D1 ──→ MPU6050 SCL

3️⃣ ADC + 분압기 (신호 경로)
   MyoWare → [분압기] → ESP32 A0

4️⃣ PWM + MOSFET (제어 신호)
   ESP32 D9 → [100Ω] → MOSFET Gate → 진동모터


[전체 부품 위치 (Flexible PCB 상)]

    위쪽 (POWER RAIL)
    ──────────────────────────────────────────
    Bat(+) ──○──ESP32──○──MYO──○──MPU──○──MOS
             [배선 경로 위]

    중앙부 (신호선)
    ──────────────────────────────────────────
    D0 ──I2C──→ MPU SDA
    D1 ──I2C──→ MPU SCL
    A0 ──ADC──→ 분압기 (MYO 신호)
    D9 ──PWM──→ MOSFET Gate

    아래쪽 (GND RAIL)
    ──────────────────────────────────────────
    Bat(−) ──●──ESP32──●──MYO──●──MPU──●──MOS
            [배선 경로 아래]


[문제 해결 체크리스트]

문제: Power Rail 단락 (배터리 소비 빠름)
→ 확인: Power와 GND 선이 접촉하지 않는가?
→ 확인: 부품 핀이 올바른 위치에 납땜되었는가?

문제: I2C 통신 실패 (MPU6050 미감지)
→ 확인: SDA/SCL 선이 바뀌지 않았는가?
→ 확인: pull-up 저항이 있는가? (3.3V ~ SDA/SCL)
→ 확인: 주소는 0x68인가? (멀티미터로 연속성 테스트)

문제: ADC 값이 이상함 (0 또는 최대)
→ 확인: 분압기 위아래가 올바르게 연결되었는가?
→ 확인: MyoWare SIG가 분압기 중점으로 연결되었는가?

문제: 진동모터가 작동하지 않음
→ 확인: MOSFET Gate가 D9로부터 신호를 받는가?
→ 확인: MOSFET Drain/Source 극성이 맞는가?
→ 확인: 진동모터 자체가 정상인가? (멀티미터로 연속성 테스트)


[납땜 온도 & 시간]

• 솔더: Pb-free (무연), 지름 1mm
• 인두 온도: 350°C
• 접촉 시간: 3~5초 (연결이 확실하고 광택이 나야 함)
• 과열 방지: 같은 포인트에 3회 이상 가열 금지


[최종 전원 인가 전 확인]

□ Power Rail 전체 연속성 (멀티미터 저항 측정: 0Ω)
□ GND Rail 전체 연속성 (멀티미터 저항 측정: 0Ω)
□ Power ↔ GND 간 단락 여부 (저항 측정: 무한대 또는 매우 높음)
□ 각 부품 핀 배향 확인 (사진 또는 스펙시트 재확인)
□ 배터리 극성 재확인 (반복 확인!)
□ 모든 납땜점 육안 검사 (광택 있고 균일한가?)
□ 이물질 제거 (플럭스, 산화물)

→ 모두 OK면, 배터리 연결 → 부팅 확인 → 센서 동작 테스트


[전류 예상값]

ESP32-S3 idle:        ~40 mA
+ MyoWare 수신 중:    ~45 mA
+ MPU6050 100Hz:      ~50 mA
+ LED ON (RGB):       ~60 mA
+ 진동모터 PWM:       ~150 mA (피크, 평균 ~50 mA)

→ 배터리 3.7V 300mAh 기준: 약 6시간 연속 동작 예상
