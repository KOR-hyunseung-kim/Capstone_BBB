# 전원 배포 회로 (배터리 & LDO)

**목적**: 3.7V 배터리 → 3.3V LDO 변환 → 센서/MCU 전원, 모터는 배터리 직결

---

## 부품 목록

| 부품 | 모델 | 수량 | 비고 |
|------|------|------|------|
| 배터리 | 3.7V Li-Po 500mAh JST 1.25mm | 1 | 메인 전원 |
| LDO | ESP32-S3 내장 | 1 | 3.7V → 3.3V 변환 |

---

## 전원 분배 원리

```
배터리 3.7V (400mAh 가정)

┌─────────────────────────────────────────┐
│           배터리 3.7V (+)               │
└─────────────┬──────────────┬────────────┘
              │              │
         [경로 A]        [경로 B]
              │              │
         ┌────▼───┐      ┌───▼───────┐
         │ LDO    │      │ 모터(직결)│
         │3.7→3.3V│      │ 3.7V      │
         └────┬───┘      └───┬───────┘
              │              │
         ┌────▼──────────────▼─┐
         │     3.3V 버스       │
         │ MCU / 센서         │
         └────────┬───────────┘
                  │
         (배터리 음극 GND와 통합)
```

## 회로도 (상세)

### 전원 입력

```
배터리 JST 커넥터:
  (+) 빨강 ─────┬─→ ESP32 BAT 핀 (LDO 입력)
                └─→ 모터(+) 직결
  
  (-) 검정 ─────┬─→ ESP32 GND (공통 기준)
                └─→ 모터(-) / MOSFET Source (공통 기준)
```

### ESP32-S3 내부 전원 구조

```
┌──────────────────────────────┐
│   ESP32-S3 DevKit 내부       │
│                              │
│  BAT 핀 (3.7V 입력)          │
│    │                         │
│    ├─→ [LDO] ─→ 3.3V        │
│    │            │            │
│    │            ├─→ MCU      │
│    │            ├─→ RAM      │
│    │            └─→ 3.3V_OUT│
│    │                   │     │
│    │    ┌──────────────┘     │
│    │    │                    │
│    ├─→ [USB Detect]          │
│    │                         │
│   GND ─→ 공통 그라운드       │
│                              │
└──────────────────────────────┘
       ↓
  [3.3V 버스]
     ├─→ MyoWare VS+
     ├─→ MPU6050 VCC
     ├─→ OLED VCC
     └─→ GPIO 풀업
```

### 배터리 지속 시간 추정

**배터리**: 500mAh  
**MCU + 센서**: ~100mA (WiFi 포함)  
**지속 시간**: 500mAh / 100mA ≈ **5시간** (모터 제외)

**목표**: 2시간 이상 → 현재 스펙 OK ✓

## 배선 상세

### 배터리 JST 커넥터 연결

```
JST 1.25mm 커넥터 (상단 보기):
  [+] 빨강 ──→ BAT 핀
  [-] 검정 ──→ GND 핀

배터리 케이블:
  빨강: (+) 양극
  검정: (-) 음극
```

### 배터리 ↔ ESP32 배선

```
배터리(+) JST 빨강 ─────[점퍼선]───→ ESP32 "BAT" 또는 "5V" 핀
                                    (또는 USB 근처 5V)

배터리(-) JST 검정 ─────[점퍼선]───→ ESP32 "GND" 핀
                                    (모든 GND와 병렬)
```

### 배터리 ↔ 모터 배선

```
배터리(+) JST 빨강 ─────[점퍼선]───→ 모터(+)
                                    (ESP32 우회, 직접 연결)

배터리(-) JST 검정 ─────[점퍼선]───→ MOSFET Source / 모터(-)
                                    (공통 GND)
```

## 펌웨어 전원 관리 (MicroPython)

### 배터리 전압 감지 (ADC 이용)

```python
from machine import ADC, Pin

# 배터리 모니터링 GPIO (예: GPIO2)
# 주의: 분압 저항 필요 (3.7V → 3.3V 이하)
# 계산: R1=4.7kΩ, R2=10kΩ → 3.7V*10/(4.7+10) ≈ 1.2V

adc_bat = ADC(Pin(2))
adc_bat.atten(ADC.ATTN_11DB)
adc_bat.width(ADC.WIDTH_12BIT)

def read_battery_voltage():
    """배터리 전압 읽기 (분압 저항 기반)"""
    raw = adc_bat.read()
    measured_v = raw * 3.3 / 4095
    # 분압 역계산: V_bat = V_measured * (R1+R2) / R2
    battery_v = measured_v * (4.7 + 10) / 10
    return battery_v

# 배터리 체크
while True:
    v = read_battery_voltage()
    print(f"Battery: {v:.2f}V")
    if v < 3.4:
        print("Low battery warning!")
```

### 절전 모드 (필수!)

```python
import machine

def deep_sleep_mode(seconds=60):
    """깊은 절전 모드 (WiFi 비활성화)"""
    machine.deepsleep(seconds * 1000)

def light_sleep_mode(seconds=1):
    """가벼운 절전 모드 (WiFi 유지)"""
    machine.lightsleep(seconds * 1000)

# 사용: 센서 샘플링 사이에 절전
```

## 배선 체크리스트

```
[ ] 배터리 JST 커넥터 극성 확인 (+ 빨강, - 검정)
[ ] 배터리(+) → ESP32 BAT 연결
[ ] 배터리(-) → ESP32 GND 연결
[ ] 배터리(+) → 모터(+) 직결 (LDO 우회)
[ ] 배터리(-) → MOSFET Source 연결 (공통 GND)
[ ] 멀티미터로 각 지점 전압 확인:
    - BAT 핀: 3.6~3.8V
    - 3.3V: 3.2~3.4V
    - GND: 0V (기준)
[ ] 배터리 연결 전 회로도 최종 확인
```

## 배터리 보호 및 유지보수

### 충전 전 확인

```
1. 멀티미터로 배터리 전압 확인
   - 3.0V 이상: 충전 OK
   - 2.8V 이하: 폐기 (리튬 손상)

2. 배터리 물리적 손상 확인
   - 팽창: 사용 금지
   - 누액: 사용 금지
   - 정상: 사용 가능

3. 충전 환경
   - 온도: 0°C ~ 45°C
   - 습도: 건조한 환경
   - 충전 시간: 1~2시간 (C/10 기준)
```

### 장기 보관

```
1. 배터리 분리 (ESP32와 분리)
2. 서늘한 곳 (15~25°C) 보관
3. 월 1회 충전 (완방지)
4. 1년 이상 보관 시 교체 권장
```

## 문제 해결

### 문제 1: ESP32가 부팅되지 않음

```
원인:
1. 배터리 전압 부족 (2.8V 이하)
2. 배터리 극성 반대
3. USB 충전 회로 고장

해결:
1. 멀티미터로 배터리 전압 확인 (3.6V 이상?)
2. 극성 다시 확인 (+ 빨강, - 검정)
3. USB-C로 직접 충전 테스트
```

### 문제 2: 센서가 동작하지 않는데 MCU는 정상

```
원인:
1. 3.3V 버스 전압 부족
2. GND 연결 불량
3. 센서 불량

해결:
1. 3.3V 출력 확인 (멀티미터)
2. 모든 GND 멀티포인트 확인
3. 센서를 다른 I2C 주소로 테스트
```

### 문제 3: 모터가 진동하지 않음

```
원인:
1. 배터리 직결 끊김
2. MOSFET 고장
3. GPIO20 출력 불량

해결:
1. 배터리(+) → 모터(+) 도통 확인
2. MOSFET 교체
3. GPIO20을 LED로 테스트
```
