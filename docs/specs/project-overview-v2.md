## 1. 문제 정의 — 국내 근골격계 질환 현황

### 통계 데이터

| 지표 | 수치 | 출처 |
|------|------|------|
| 근골격계 질환 산업재해 비율 | 전체 직업병의 **약 70%** | 고용노동부 산업재해 현황분석 |
| 연간 근골격계 질환 산재 승인 건수 | ~9,000건 (2022년 기준) | 안전보건공단 |
| 물류·운반업 근골격계 질환 발생률 | 타 직군 대비 **3.2배** | 한국산업안전보건연구원 |
| 전완 건염(Forearm Tendinitis) 재발률 | 초기 부상 후 **60% 재발** | 국내 정형외과 메타연구 |
| 근골격계 질환 1인당 산재 처리 비용 | 평균 **₩12,000,000** | 근로복지공단 |

### 핵심 문제: "피로 → 부상" 연결 고리

```
반복 작업 → 근피로 누적 → 순간 근력 저하 → 부하 집중 → 부상
                ↑
         [감지 수단 없음]
         작업자 본인도 모름
```

현존 문제:
- 근피로도는 주관적 감각에만 의존 → 개인차 크고 과소 신고됨
- 경고 신호 없이 임계치 돌파 → 갑작스러운 부상
- 사후 치료보다 예방이 비용 효율 10배 이상 높음

## 2. 타겟 페르소나

### 페르소나 A — 물류 창고 작업자

> **김지훈 (28세)** — 수도권 물류 센터 근무 3년차  
> 하루 8시간, 박스 반복 하역 작업. 평균 들기 횟수 500회/일, 평균 하중 15kg.

**하루 일과 (Before BBB)**

| 시간 | 상황 |
|------|------|
| 09:00 | 작업 시작. 전완근 상태 양호 |
| 10:30 | 피로 느끼기 시작. 그러나 "조금만 더" 판단 |
| 11:00 | 15kg 박스 들어올리다 손목에 찌릿함. 그냥 무시 |
| 13:00 (1개월 반복 후) | 전완 건염 진단. 3주 병가 |

**왜 기존 방법으로 해결 안 되나**
- 운동 기록 밴드(갤럭시 핏 등): 가속도 센서 기반, 팔의 움직임만 측정. 근육 내 피로 상태 감지 불가
- 작업자 본인 판단: 피로도는 느끼는 시점이 이미 80% 이상 소진된 후 → 대응 타이밍 놓침
- 관리자 육안 모니터링: 100명 작업장에서 개인별 피로도 추적 불가능

**After BBB**

| 시간 | 상황 |
|------|------|
| 09:00 | BBB 착용, Safety Mode 활성화 |
| 10:25 | 전완근 Median Frequency 84% → **진동 1회 + LED 노란색** |
| 10:27 | 자발적 5분 휴식 (동료에게 티내지 않고 자연스럽게) |
| 11:40 | 두 번째 경보 → 10분 휴식 |
| 18:00 | 퇴근. 전완 통증 없음 |

BBB 착용 시: 임계치 도달 전 2~3회 선제적 휴식 → 부상 위험 대폭 감소

### 페르소나 B — 항공 정비 기술자

> **이수연 (34세)** — 국내 항공사 MRO(정비) 센터 7년차  
> 항공기 기체 점검 작업. 양손은 항상 공구 또는 부품을 잡고 있음.  
> 작업 중 노트북 매뉴얼(AMM) 확인이 수십 회 필요.

**작업 중 매뉴얼 조회 흐름 (Before BBB)**

```
공구 잡고 작업 중
  → 매뉴얼 확인 필요
  → 공구 내려놓기 (장갑 벗기 경우도 있음)
  → 터치패드/마우스로 스크롤·클릭
  → 다시 장갑 착용, 공구 집어들기
  → 작업 재개
  
소요 시간: ~15초 / 회, 하루 50회 = 12분 30초 손실
```

**왜 기존 방법으로 해결 안 되나**
- 음성인식: 항공기 격납고는 소음 85dB 이상 → 오인식률 높음. 기술 용어 인식률 저조
- 풋 스위치: 사다리 위, 협소 공간에서 발 동작 불가
- 터치스크린: 글러브 착용 시 반응 없음
- Eye Tracker: 고가(50만원↑), 장시간 집중 작업 시 눈 피로 급증

**After BBB**

```
공구 잡고 작업 중
  → 매뉴얼 확인 필요
  → 팔 오른쪽으로 기울임 → 커서 이동 (MPU6050 pitch/roll)
  → 주먹 한 번 쥐기 → 클릭 (MyoWare EMG spike 감지)
  → 작업 재개 (공구, 장갑 그대로)
  
소요 시간: ~2초 / 회
```

Control Mode 전환: BBB의 측면 버튼 1회 누름 (Safety ↔ Control)

| 방법 | 근피로 감지 | 핸즈프리 입력 | 가격 | 현장 적용성 | 비고 |
|------|-----------|------------|------|-----------|------|
| 피트니스 밴드 (미밴드, 갤럭시 핏) | ❌ 불가 | ❌ 불가 | ~₩50,000 | 보통 | 가속도 센서만, EMG 없음 |
| 임상용 EMG 장비 | ✅ 가능 | ❌ 불가 | 수백만원 | ❌ 현장 불가 | 병원 전용, 유선, 대형 |
| 스마트워치 (Apple Watch 등) | ❌ 불가 | ❌ 불가 | ~₩500,000 | 보통 | 심전도 있으나 근전도 없음 |
| 음성 인식 헤드셋 | ❌ 불가 | △ 제한적 | ~₩100,000 | ❌ 소음 환경 | 오인식, 개인정보 우려 |
| 풋 스위치 | ❌ 불가 | △ 제한적 | ~₩30,000 | ❌ 이동·협소 공간 | 고정 위치에서만 |

| **BBB** | ✅ 가능 | ✅ 가능 | ~₩50,000 | ✅ 높음 | 아대형, 경량, WiFi |

## 4. Safety Mode + Control Mode를 한 제품에 넣는 이유

### 하드웨어 재활용 — 추가 비용 없음

```
MyoWare 2.0 (전완근 EMG)
  ├── Safety Mode: Median Frequency 분석 → 피로도 판정
  └── Control Mode: Spike 감지 → 클릭 이벤트

MPU6050 (IMU)
  ├── Safety Mode: 자세 교정 보조 (선택적)
  └── Control Mode: pitch/roll → 커서 델타 X/Y
```

같은 센서를 두 가지 용도로 쓰기 때문에 **하드웨어 추가 없이** 두 기능 제공.

### 사용 맥락의 자연스러운 전환

작업자는 하루 중 두 가지 필요가 모두 생김:
- **집중 반복 작업 시간**: Safety Mode → 피로도 모니터링
- **매뉴얼 조회 순간**: Control Mode → 핸즈프리 입력

모드 전환: 측면 버튼 1회 누름 (1초 미만) → **장비 교체 없이 즉시 전환**

### 제품 포지셔닝 차별화

단일 기능 제품들의 공백:
- 피로도 모니터링만 → 틈새 시장, 범용성 낮음
- 핸즈프리 입력만 → 기존 제품(Eye Tracker, 음성인식) 경쟁 불리

**두 기능의 조합** → 산업 현장 특화 웨어러블로 명확한 포지션

## 6. 시나리오 B — Control Mode 타임라인

**배경**: 항공기 착륙장치 점검 작업, 매뉴얼(AMM) Chapter 32 확인

| 상황 | 동작 | 결과 |
|------|------|------|
| 볼트 토크 값 확인 필요 | BBB 측면 버튼 1회 → Control Mode | LED 파란색으로 전환 |
| 커서를 AMM 검색창으로 이동 | 전완 우측 기울임 (pitch +15°) | 커서 오른쪽으로 이동 |
| 검색창 클릭 | 주먹 빠르게 쥐기 (EMG spike) | 마우스 좌클릭 |
| 다음 페이지 스크롤 | 전완 아래 기울임 (roll -10°) + 주먹 | 스크롤 다운 |
| 값 확인 완료 | BBB 측면 버튼 1회 → Safety Mode | LED 초록색으로 복귀 |
| 전체 소요 | ~3초 | 공구·장갑 그대로 유지 |

## 7. 기대 효과 & 차별화 요약

### 정량 목표

| 항목 | 목표 |
|------|------|
| 근피로 임계치 초과 빈도 | 착용 전 대비 50% 감소 |
| 매뉴얼 조회 전환 시간 | 15초 → 2초 (87% 단축) |
| 디바이스 무게 | 50g 이하 (배터리·센서 포함) |
| 연속 동작 시간 | 2시간 이상 (500mAh 기준 ~2.7h) |
| EMG spike 오검출률 | < 5% |
| WiFi 센서→노트북 레이턴시 | < 50ms |

### 차별화 포인트

1. **저가 EMG 실시간 모니터링**: 기존 임상 장비 대비 1/100 비용
2. **듀얼 모드**: 한 장치로 피로도 감지 + 핸즈프리 입력 모두 제공
3. **착용 편의성**: 아대형 밴드, 50g 이하, 배터리 내장
4. **노트북 연동**: 별도 앱 불필요, Python 스크립트로 즉시 사용 가능
5. **개방형 플랫폼**: WiFi + pyautogui 구조로 다양한 애플리케이션 확장 가능
