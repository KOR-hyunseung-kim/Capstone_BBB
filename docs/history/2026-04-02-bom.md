# H/W BOM 작성 및 아키텍처 결정

**날짜**: 2026-04-02
**작업자**: hw-architect agent / Human
**관련 파일**: `docs/bom.md`, `CLAUDE.md`

## 요약
BBB 프로젝트 초기 H/W 구매 목록(BOM) 작성 완료.
모듈 단위 직결 와이어링 방식으로 확정, 별도 회로 구성 불필요.

## 결정 사항

### 아키텍처
- **노트북 연산 방식** 채택: 센서 Raw 데이터 → WiFi → 노트북 Python 처리 → 제어 명령 → ESP32 → IO 장치
- BLE HID 미사용: Control Mode도 노트북 `pyautogui`로 커서/클릭 제어
- ESP32는 Sensor Hub + IO Controller만 담당

### H/W 구성 결정
| 항목 | 결정 | 이유 |
|------|------|------|
| MCU | XIAO ESP32-S3 | WiFi 통신 필수, BLE 내장 |
| EMG | MyoWare 2.0 (×2) | 신호 품질 최우선, 스냅형 전극 |
| IMU | MPU6050 GY-521 (×3) | 검증됨, 라이브러리 풍부 |
| IMU (비교용) | BMI270 (×1) | 저전력 설계 단계 재검토용 |
| 배터리 | 500mAh JST 1.25mm | 300mAh로 2h 미달, 500mAh 선택 |
| 진동모터 | 코인 Ø10mm bare + **MOSFET 회로** | IRLML6244 + 1N4148 + 100Ω, 3소자로 경량화 |
| 3D 프린팅 | **제외** | 와이어링 조립으로 대체 |
| 브레드보드 | **제외** | 실제 디바이스에 미사용 |
| MOSFET/저항 등 | **제외** | 드라이버 내장 모듈로 대체 |

### 중요 기술 메모
- XIAO ESP32-S3 배터리 커넥터: **JST 1.25mm** (PH 2.0 아님)
- BMI160 공식 단종 (2025년 4월) → 신규 설계 사용 금지
- MyoWare 2.0 Adafruit 단종 → SparkFun DEV-21267 또는 Amazon.co.jp 구매

### 예산
- 총 예상: ~₩232,000 (MyoWare 2.0 ×2가 52% 차지)
- 해외 주문(MyoWare): 1~2주 배송 예상 → 가장 먼저 주문 필요
